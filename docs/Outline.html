Outline and Plan for RHESSys manual

Author

Janet Choate

Project Summary / Abstract

The purpose of this project is to improve the usability and accessibility of the Regional Hydro-Ecologic Simulation System (RHESSys) by developing comprehensive documentation and training materials. These materials are intended to support California Air Resources Board (CARB) staff, researchers, and the broader RHESSys modeling community in efficiently learning and applying the model.

By reducing the learning curve associated with RHESSys, these training materials will allow users to focus more on scientific analysis of model results rather than model setup and execution.

RHESSys is a key tool used to project carbon dynamics in forests, shrublands, and grasslands. However, its complexity presents a significant barrier for new users. This project addresses that challenge by developing a user manual and accompanying video tutorials that demonstrate common modeling workflows. All materials will be publicly available and designed to support both CARB staff and the broader RHESSys user community.

Scope of Work

Objective

Develop high-quality documentation and video training materials that reduce barriers to learning and applying RHESSys, with a focus on CARB-specific modeling needs.

Work to Be Performed

This project will produce two primary training resources:

1. RHESSys Training Manual

A comprehensive, standalone manual that includes:

  • Step-by-step instructions
  • Guided tutorial exercises
  • Example datasets

The manual will be designed so that a user with limited programming experience can independently set up and run RHESSys for CARB-related applications.

2. Training Videos

A set of videos aligned with topics covered in the manual that:

  • Demonstrate key workflows
  • Walk through tutorial exercises
  • Provide visual guidance for common tasks

All materials will be publicly accessible and designed for both CARB staff and external users, including graduate students and researchers.

Methodology

Data Collection and Preparation

  • Compile spatial datasets (e.g., DEM, soils, land cover)
  • Climate data acquisition and formatting
  • Develop a standardized example dataset for training

Workflow Development

  • Create preprocessing workflows for RHESSys inputs
  • Develop simulation and analysis workflows
  • Sensitivity analysis and calibration approaches
  • Ensure all processes are reproducible using R

Output Interpretation

  • Develop approaches for visualizing and interpreting model outputs
  • Provide reproducible examples for evaluating model performance

Expected Outcomes

  • Fully developed RHESSys training manual
  • Suite of instructional videos
  • Publicly accessible example dataset for tutorials
  • Reduced time required for new users to learn/apply RHESSys
  • Increased adoption and consistency in RHESSys applications
  • Reproducible modeling workflows

Potential Challenges and Mitigation Strategies

Challenge Mitigation Strategy
Steep learning curve Structured tutorials and progressive exercises
Data complexity Develop a standardized example dataset
Software dependencies Use open-source tools and clear installation instructions
User experience/knowledge Design materials that support a range of skill levels

Benchmarks for Success

Success will be evaluated based on:

  • Completion of all deliverables within the project timeline
  • Positive feedback from CARB staff
  • Ability of new users to independently run RHESSys using the provided materials
  • Public availability and usability of documentation, datasets, and videos

Information and Documentation Requirements

All materials developed in this project will:

  • Be fully documented and reproducible
  • Support independent use without contractor assistance
  • Meet accessibility standards (WCAG 2.0 AA)
  • Use open-source tools and permissive licensing
  • Avoid reliance on paid platforms

All datasets, workflows, and documentation will be delivered in a clean, well-organized format suitable for long-term use and transferability.

Training Materials Structure (Manual Outline)

Part 1: Introduction and Setup

  • Overview of RHESSys
  • Installation (MacOS, Linux, Docker)
    • also address Singularity Container Platform using a Singularity Image Format (.sif file) for running RHESSys on a cloud computing service (AWS)

Part 2: Data and Preprocessing

  • Input data requirements
    • Spatial Data
    • Climate
    • Definition Files - parameter configuration files that define the physical, biological, and hydrological characteristics of spatial units (soil, vegetation, landuse)
    • Worldfile
      • Hierarchical spatial containment structure
      • State variables and linking identifiers
    • Flowtable
      • Patch connectivity
      • movement of water and nutrients across the landscapte
    • Tecfile (Temporal Event Control file)
      • Manage timing of output printing
      • Control simulation events (schedule when non-continuous events occur, i.e. fire, land use change)
      • Trigger a worldfile redefinition to update properties of the landscape
    • Output filters - specify criteria for selecting and filtering the output data generated by a RHESSys simulation
      • Time interval (i.e. daily, monthly)
      • Spatial level (i.e. Basin, patch)
      • Particular processes
  • Preprocessing
    • Spatial processing using WhiteboxTools and R
    • Worldfile and flowtable generation
      • Worldfile template
      • RHESSysPreprocessing Tools

Part 3: Running RHESSys

  • Model configuration - RHESSysIOinR tools
  • Executing simulations
  • Output handling

Part 4: Calibration and Sensitivity Analysis

  • Parameter selection
  • Calibration workflows
  • Performance metrics

Part 5: Advanced Features

  • Disturbance modeling (fire, drought)
  • Vegetation change
  • Multi-scale routing
  • Target-driven spin-up

Part 6: Appendices

  • Command-line reference
  • Code structure
  • Additional figures

Resource Sharing and Accessibility

All materials will be:

  • Hosted online (e.g., GitHub Pages)
  • Available as downloadable datasets and PDFs
  • Accompanied by video tutorials
  • Designed for long-term public access

Review and Approval Process

All outputs—including publications, presentations, and media—will undergo CARB review prior to release. CARB will provide timely feedback to ensure alignment with project goals and standards.
