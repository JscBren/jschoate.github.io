Outline and Plan for RHESSys manual

Author

Janet Choate

Project Summary

The purpose of this project is to improve the usability and accessibility of the Regional Hydro-Ecologic Simulation System (RHESSys) by developing comprehensive documentation and training materials. These materials are intended to support California Air Resources Board (CARB) staff, researchers, and the broader RHESSys modeling community in efficiently learning and applying the model.

By reducing the learning curve associated with setting up a RHESSys modeling project, these training materials will allow users to focus more on scientific analysis of model results rather than model setup and execution.

Scope of Work

Objective

Develop high-quality documentation and video training materials that reduce barriers to learning and applying RHESSys, with a focus on CARB-specific modeling needs.

Work to Be Performed

This project will produce two primary training resources:

1. RHESSys Training Manual

A comprehensive, standalone manual that includes:

  • Step-by-step instructions
  • Guided tutorial exercises
  • Pre-built example datasets

The manual will be designed so that a user with limited programming experience can independently set up and run RHESSys for CARB-related applications.

2. Training Videos

A set of videos aligned with topics covered in the manual that:

  • Demonstrate key workflows
  • Walk through tutorial exercises
  • Provide visual guidance for common tasks

All materials will be publicly accessible and designed for both CARB staff and external users, including graduate students and researchers.

Methodology

Installation

  • RHESSys Compute and Execution Environments
  • Dependencies
  • Workflow and Analysis Tools

Data Collection and Preparation

  • Compile spatial datasets (e.g., DEM, soils, land cover)
  • Climate data acquisition and formatting
  • Develop a standardized example dataset for training

Workflow Development

  • Create preprocessing workflows for RHESSys inputs
  • Develop simulation and analysis workflows
  • Spin-up, sensitivity analysis and calibration approaches
  • Ensure all processes are reproducible using R

Output Interpretation

  • Develop approaches for processing, visualizing and interpreting model outputs
  • Provide reproducible examples for evaluating model performance

Expected Outcomes

  • Fully developed RHESSys training manual
  • Suite of instructional videos
  • Publicly accessible example dataset for tutorials
  • Reduced time required for new users to learn/apply RHESSys
  • Increased adoption and consistency in RHESSys applications
  • Reproducible modeling workflows

Benchmarks for Success

Success will be evaluated based on:

  • Completion of all deliverables within the project timeline
  • Positive feedback from CARB staff
  • Ability of new users to independently run RHESSys using the provided materials
  • Public availability and usability of documentation, datasets, and videos

Information and Documentation Requirements

All materials developed in this project will:

  • Be fully documented and reproducible
  • Support independent use
  • Comply with accessibility standards (WCAG 2.0 AA)
  • Use open-source tools and permissive licensing
  • Avoid reliance on paid platforms
  • Be clearly organized for long-term use and transferability

    • Overview of RHESSys

      • What it is used for/What it simulates
    • Model capabilities and limitations
    • Prerequisites and requirements
    • RHESSys workflow overview
    • Key concepts and terminology (Frequently used terms)
      • Spatial hierarchy (basin, hillslope, zone, patch, stratum)
      • File types (Worldfile, Flowtable, Tecfile, Parameter/Climate files)
      • State variables (store quantities) and fluxes
      • Spatial/Temporal step
      • Inputs (i.e., climate, parameters)
      • Outputs (i.e., streamflow, LAI)
    • Manual structure (navigation, organization - links to each section)
    • References and Additional Resources
    • Quick start guide
  2. Installation
    • Setup Options (description, who should use/recommendation)
      • Local Installation (MacOS, Linux)
      • Containerized setup (Docker)
      • HPC/Cloud (SIF/Singularity)

      • Local
        • System Dependencies and Compilation Requirements
          • Terminal - command line access
          • Privileges (i.e. Administrator access)
          • Compilers/Build tools (gcc, clang, make)
          • Libraries (i.e. NetCDF)
          • Installation steps:
            • MacOS - Homebrew, command examples
            • Linux - command examples
        • Installing RHESSys code
          • Clone repository
          • Compile
          • Verify installation
          • Shell configuration (adding to path)
          • Common issue and solutions
      • Docker
        • Requirements
        • Steps
      • SIF
        • Requirements
        • Steps
    • Workflow & Analysis Tools
      • R/Rstudio & packages (i.e., Tidyverse)
      • WhiteboxTools with R
      • RHESSys R packages
        • RHESSysPreprocessing
        • RHESSysIOinR

Part 2: Data Preparation and Preprocessing

  • Input data requirements
    • Spatial Data
      • Required
        • DEM
          • resolution
          • reprojection
        • Waterhsed basin
        • Slope/Aspect
        • Sub-basin structure
        • Stream network (gage location)
        • Patch structure
          • approaches
      • Recommended
        • Horizon maps
        • vegetation
        • soils/soil attributes
        • land cover history
        • climate station locations (if applicable)
    • Climate Data
      • Input Formats (point, grid, NetCDF)
      • Required meteorological variables
      • Optional meteorological variables
      • Base station
    • Worldfile
      • Hierarchical spatial containment structure
      • State variables and pointers (linking identifiers)
      • Template
      • Header
      • Required and optional state variables
    • Flowtable
      • Patch connectivity
      • Lateral processes
    • Definition (default) Files
      • Eco-physiologic, hydro-geologic, land-use, and meteorologic parameters
      • Base files & Default values
    • Tecfile (Temporal Event Control file)
      • Manage timing of output printing
      • Control simulation events (schedule when non-continuous events occur, i.e. fire, land use change)
      • Trigger a worldfile redefinition to update properties of the landscape
    • Output filters - specify criteria for selecting and filtering the output data
      • Filter rules
      • Filter conditions
      • Processes associated with temporal interval and spatial level
        • sub-level output
        • sub-routine output
  • Preprocessing
    • Standard directory structure and naming convention
    • Spatial processing using WhiteboxTools and R
    • Worldfile and flowtable generation
      • Required rasters
      • Editing the worldfile template
      • RHESSysPreprocessing R package

Part 3: RHESSys Implementation and workflow

  • Model configuration via RHESSysIOinR tools
    • Fuctions to provide the inputs necessary to run RHESSys
  • Output filter design
    • Tecfile/Output Filter interaction for expected output
    • Selecting RHESSys output variables
      • Focus on variables relevant to your research questions and analysis goals
      • Reassigning/Combining variables
  • Executing simulations in R
    • Single run
    • Multiple runs
  • Post-processing vi RHESSysIOinR tools
    • Functions for integrating, synthesizing, and visualizing spatiotemporal output

Part 4: Initialization, Calibration & Sensitivity Analysis

  • Initialization approaches (Spin-up to a state of equilibrium)
  • Sensitivity Analysis and Parameter Selection (identifying influential variables)
  • Parameter Space Characterization (establishing realistic bounds/ranges)
  • Calibration Frameworks and Optimization Workflows
  • Objective Functions and Performance Metrics

Part 5: Advanced Features

  • Disturbance modeling
    • drought induced mortality
    • fire spread and fire effects
  • Land Cover/Vegetation change
    • fuel treatments - thinning, controlled burns
  • Multi-scale routing - fine-scale heterogeneity
  • Target-driven spin-up (run to reach spatially explicit targets, i.e. LAI)

Part 6: Appendices

  • Command-line reference
  • Code structure
  • Additional figures
